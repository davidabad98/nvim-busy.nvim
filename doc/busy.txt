1. INTRODUCTION                                                   *busy-intro*

nvim-busy shows a small animated braille indicator at the bottom of the
screen (a floating window, not a statusline component) whenever Neovim is
waiting on an asynchronous operation:

  - LSP server background indexing / loading ($/progress messages)
  - LSP navigation requests: gd, gD, gi, gr, K, gk
  - Telescope live_grep / find_files search in progress
  - User entering command mode (`:`, `/`, `?`)

The indicator disappears automatically when the operation completes.
It does not steal focus and does not interfere with normal editing.

A single |vim.uv| timer at 80 ms drives the animation loop. Signal sources
are event-driven (autocmds + callbacks), never polling.

==============================================================================
2. REQUIREMENTS                                            *busy-requirements*

  - Neovim >= 0.10
  - No other plugins required

Optional integrations:
  - telescope.nvim  — animated in-prompt spinner via get_status_text

3. SETUP                                                       *busy.setup()*

Call setup() once during Neovim startup, typically from your lazy.nvim spec:
>lua
  require("busy").setup(opts)
<
`opts` is an optional table; omitted keys use their defaults.
See |busy-config| for all available options.

Recommended lazy.nvim spec: >lua
  {
    "davidabad98/nvim-busy.nvim",
    lazy     = false,   -- eager load so autocmds fire from startup
    priority = 900,     -- load before LSP / Telescope
    opts     = {},
  }
<

4. API                                                            *busy-api*

busy.push({id})                                               *busy.push()*
  Mark a named task as started. The indicator appears immediately.
  `id` must be a non-empty string identifying the busy reason. Multiple
  concurrent callers with different ids are supported; the indicator stays
  until all have called pop().

busy.pop({id})                                                 *busy.pop()*
  Mark a named task as finished. The indicator disappears when all pushed
  ids have been popped.
  `id` must match the string passed to |busy.push()|. Calling pop() with an
  id that was never pushed is a no-op.

busy.wrap({fn} [, {id}])                                      *busy.wrap()*
  Wrap a function with automatic push/pop so the indicator shows for the
  duration of the call. Designed for wrapping vim.lsp.buf.* in on_attach: >lua

    map("n", "gd", busy.wrap(vim.lsp.buf.definition, "lsp:gd"),
        { buffer = bufnr, desc = "Goto Definition" })
<
  `fn`  — the function to wrap (e.g. vim.lsp.buf.definition).
  `id`  — optional stable id string; defaults to a function-address-based key.

  A safety TTL (|lsp.request_ttl_ms|) always clears the id even if the
  server never responds (crash, symbol not found, etc.).

5. CONFIGURATION                                               *busy-config*

Default configuration with all options shown: >lua

  require("busy").setup({
    animation = "dots",        -- "dots"|"arc"|"line"|"bounce"|string[]
    speed_ms  = 80,
    position  = "bottom-left", -- "bottom-left"|"bottom-right"
    text      = " loading",    -- suffix label, or false to show only the frame
    blend     = 0,             -- winblend 0-100
    zindex    = 45,
    lsp = {
      enabled        = true,
      watch_progress = true,
      watch_requests = false,
      request_ttl_ms = 10000,
    },
    telescope = {
      enabled         = true,
      animate_counter = true,
    },
    cmdline = {
      enabled   = true,
      patterns  = { ":", "/", "?" },
      blend     = 30,
      animation = "dots",
      speed_ms  = 80,
    },
  })
<

animation ~
  Animation pattern name or a custom table of frame strings.
  Built-in patterns: "dots", "arc", "line", "bounce"
  Custom example: `animation = { "▏", "▎", "▍", "▌", "▋", "▊", "▉", "█" }`

speed_ms ~
  Milliseconds between animation frames. Default: 80.

position ~
  Where the floating bar appears. "bottom-left" or "bottom-right".

text ~
  String appended after the spinner frame (e.g. " loading"). Set to `false`
  to display only the spinner character with no label.

blend ~
  `winblend` of the floating window. 0 = fully opaque background.
  100 = fully transparent (only the text characters are visible).

zindex ~
  Z-order of the floating window. Default 45 places it above normal editor
  windows but below completion menus (zindex 100).

lsp.watch_progress ~
  When true, the indicator shows during LSP $/progress messages
  (background indexing, server loading). Uses the LspProgress autocmd
  (Neovim >= 0.10).

lsp.watch_requests ~
  When true, the indicator shows while waiting for LSP server responses to
  navigation requests (gd, gr, K, etc.). Requires wrapping the keymaps via
  |busy.wrap()| in on_attach. Default: false.

lsp.request_ttl_ms ~
  Fallback timeout in milliseconds for request wrapping. The indicator is
  always cleared after this duration even if the server never responds.
  Default: 10000 (10 seconds).

telescope.enabled ~
  When true, hooks Telescope's TelescopeFindPre event so the bottom bar
  activates while a picker is open and searching. Default: true.

telescope.animate_counter ~
  When true, the get_status_text function exported from busy.telescope
  replaces Telescope's built-in "*" marker. Must be wired in manually —
  see |busy-telescope|.

cmdline.enabled ~
  When true, a full-screen dim overlay and a centred animated spinner are
  shown whenever the user enters command mode (`:`, `/`, `?`). Both
  disappear on CmdlineLeave. Default: true.
  See |busy-cmdline| for full details.

cmdline.patterns ~
  List of cmdline types that trigger the overlay. Matches the pattern
  argument of the CmdlineEnter autocmd (single characters). Default:
  { ":", "/", "?" }.

cmdline.blend ~
  winblend of the full-screen dim overlay. 0 = opaque, 100 = invisible.
  30 gives a legible but clearly-dimmed effect. Default: 30.

cmdline.animation ~
  Animation pattern for the centred spinner. Same values as the top-level
  `animation` option. Default: "dots".

cmdline.speed_ms ~
  Milliseconds between spinner frames. Independent of the top-level
  `speed_ms`. Default: 80.

6. HIGHLIGHT GROUPS                                          *busy-highlights*

Two highlight groups are defined at setup() time. They use `default = true`
so they are never forcibly overridden by the plugin after a colorscheme change.

BusyIndicator ~
  Applied to the entire floating window via `winhighlight = "Normal:BusyIndicator"`.
  Controls the background colour and the spinner character's foreground colour.
  Default: links to |hl-Comment|.

BusyIndicatorText ~
  Applied via extmark to the label portion only (the " loading" text).
  Default: links to |hl-Comment|.

BusyCmdlineOverlay ~
  Applied to the full-screen dim overlay window via
  `winhighlight = "Normal:BusyCmdlineOverlay,EndOfBuffer:BusyCmdlineOverlay"`.
  Default: `{ bg = "#000000" }` (black) — combined with winblend=30 this
  creates the dimming effect.

BusyCmdlineSpinner ~
  Applied to the centred spinner window.
  Default: links to |hl-Statement| (typically a bright colour) so it is
  visible against the dimmed overlay.

Override example: >lua
  vim.api.nvim_set_hl(0, "BusyIndicator",      { fg = "#cba6f7", bg = "#1e1e2e" })
  vim.api.nvim_set_hl(0, "BusyIndicatorText",  { fg = "#a6adc8", bg = "#1e1e2e" })
  vim.api.nvim_set_hl(0, "BusyCmdlineOverlay", { bg = "#000000" })
  vim.api.nvim_set_hl(0, "BusyCmdlineSpinner", { fg = "#cba6f7" })
<
Place this after setup() or inside an |autocmd| for ColorScheme.

7. TELESCOPE INTEGRATION                                     *busy-telescope*

7.1 Bottom bar ~
  Enabled automatically when `telescope.enabled = true` (the default). The
  bottom bar activates when a Telescope picker opens and deactivates when the
  picker is closed or the search completes.

7.2 In-prompt animated counter ~
  Replace Telescope's built-in "*" in-progress marker with an animated
  spinner by adding `get_status_text` to your Telescope defaults: >lua

    opts = {
      defaults = {
        get_status_text = function(self, opts)
          local ok, bt = pcall(require, "busy.telescope")
          if ok then return bt.get_status_text(self, opts) end
        end,
      },
    },
<
  While searching: shows "<frame>  N / M" (frame advances with time).
  When done:       shows "N / M" (plain count, spinner gone).
  With selection:  shows "<frame>  S / N / M" (selected / showing / total).

8. COMMANDS                                                  *busy-commands*

:BusyStatus                                                   *:BusyStatus*
  Print the current set of active task ids to |vim.notify|.
  Shows "idle" when no tasks are active, or a table of id → count pairs
  when one or more tasks are in progress. Useful for debugging stale state.

9. COMMAND MODE OVERLAY                                        *busy-cmdline*

When `cmdline.enabled = true` (the default), entering `:`, `/`, or `?`
causes two floating windows to appear over the editor content area:

1. Full-screen dim overlay ~
   Covers the entire content area (not the statusline or cmdline row).
   Uses `relative="editor"` with `zindex=10` and `winblend` from
   `cmdline.blend` (default 30).  Highlight group: |BusyCmdlineOverlay|.

2. Centred animated spinner ~
   A 3-cell-wide, 1-line float centred in the content area, showing a
   single braille animation frame.  Uses `zindex=11`.
   Highlight group: |BusyCmdlineSpinner|.

Both windows are destroyed on `CmdlineLeave`.  A dedicated |vim.uv| timer
drives the spinner; it starts on enter and stops on leave, so there is zero
CPU cost when command mode is not active.

The feature responds to |VimResized| and repositions both windows whenever
the terminal is resized while command mode is active.

Configuration keys — see |busy-config| (cmdline.*):
  - cmdline.enabled   — enable/disable the feature
  - cmdline.patterns  — list of cmdline types to match
  - cmdline.blend     — overlay transparency
  - cmdline.animation — animation pattern for the spinner
  - cmdline.speed_ms  — spinner tick interval

Highlight groups — see |busy-highlights|:
  - |BusyCmdlineOverlay| — controls the dim tint colour
  - |BusyCmdlineSpinner| — controls the spinner foreground colour
